分析過程

[markup]
#  defaultMarkdownHandler = "pandoc"
#  defaultMarkdownHandler = "blackfriday"
  defaultMarkdownHandler = "goldmark"

在上面的三個選項中,只有pandoc 有作用(而且.Rprofile 中的 options 不能是blogdown.method = 'markdown')。

附上md內容

```r 🏷 甚麼都沒有  
    
  
1:300
#>   [1]   1   2   3   4 --- 以下省略 ---
```

測試結果摘要

  • ❌blackfriday的時候
    • class.output 的設定,完全打亂
  • ❌goldmark 雖然沒有完全打亂,但也是不行。
  • ✔pandoc + method=markdown的組合都沒問題。
  • 🏷只要是method=custom 都不行。

❌錯誤範例1 沒有下面這種寫法

```r, x 
this is code 
```
=> 翻譯成html(整個被當成一個語言`r, x`,highlight 無法作用),下面也不行。
```r x
this is code
```

一些網路上的討論

在rmarkdown中，設定class.source="bg-danger"的意思是把輸入code 的類別設定為bg-danger。參考下面的設定,一個是給輸入,一個是給輸出

  • class

    • source

    • output

      ```{r class.source="bg-danger", class.output="我在這裡"}
mtcars[1:5, "mpg"]
```

執行

#> [1] 21.0 21.0 22.8 21.4 18.7

執行結果(從md檔案剪出來的) - 當method=custom的時候markup blackfriday,goldmark,pandoc都不出來任何類別,如下:

```
#> [1] 21.0 21.0 22.8 21.4 18.7
```

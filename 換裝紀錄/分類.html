實驗註記

uglyurls=false

如果設定 uglyurls=false,並且面板上有tags: ["xx"]則,輸入http://localhost/tags 會顯示列表。

但是如果自訂taxonomies

  • 參考 例如,我在config.toml中加入下面的設定

    [taxonomies]
series = "series"

    ❓此時有問題: localhost:1313/tags失效。也就是說,HUGO預設你關掉了預設tags,categories,因此必須自己處理。

💡怎樣處理?

  1. 再自己加上

    [taxonomies]
tag = "tags"
category = "categories"
series = "series"

  2. 以series為例，增加模板 layouts/_default/series.terms.html，內容範例

    {{ define "main" }}

<div class="content">
<div class="article-wrapper u-cf single">
    <a class="bubble" href="{{ "/series/" | relURL }}">
        <i class="fa fa-fw fa-list-alt"></i>
    </a>

    <article class="article">
        <div class="content">
            <h3>{{ i18n "series" | default "series" }}</h3>
            <hr>
            <ul id="all-series">
                {{ range $name, $taxonomy := .Site.Taxonomies.series }}
                    {{ $series_name := $name }}
                    {{ $series_path := (printf "/series/%s" (urlize $name)) }}
                    {{ $series_page := site.GetPage $series_path }}

                    {{ if $series_page }}
                        {{ $series_name = $series_page.Title }}
                    {{ end }}

                    <li><a href="{{ $series_path }}">{{ $series_name }} ({{ $taxonomy.Count }})</a></li>
                {{ end }}
            </ul>
        </div>
    </article>
</div>
</div>
{{ end }}

如何拿到正確tags所在網頁的網址

這是前一版的partial模板(tags.html),用來顯示目前md中標籤。 缺點是必須要判斷是否uglyurls的設定真假值。但是HUGO不讓我們拿(.Site.uglyURLs 會抱怨沒有這個欄位)

{{ if .Params.tags }}
<div class="tags">
{{range .Params.tags}}
  {{- if (and (isset $.Site.Params "uglyurls") ($.Site.Params.uglyURLs)) -}}
    <a class="tag-link" href="/tags/{{ . | urlize }}.html">{{ . }}</a>   
  {{- else -}}
    <a class="tag-link" href="{{ "/tags/" | relURL }}{{ . | urlize }}">{{ . }}</a> 
  {{- end -}} 
{{end}}
</div>
{{end}}

改進(但是這裡是短碼,上面是partial)


{{ $data := .Site.Taxonomies.tags }}

{{range $.Page.Params.tags}}
{{ $any := index $data . }}
  {{range $any}}
  <li><a href={{.RelPermalink}}>{{.Title}}</a> </li>
  {{end}}
{{end}}

  • 在這個短碼中,有些怪問題

    • $.Page 等於 .Page
    • ❓ .Page.Params❌ 不對 , $.Page.Params✔ 可以,就是前面要加$.

    • 在寫上面的短碼的時候,我參考這裡,❓但是下面的.Data在上面partial(tags.html)無效❓。

      <ul class="tags-listing">
	    {{ $data := .Data }}
	    {{ $thisURL := .URL}}
	    {{ range $key,$value := .Data.Terms.Alphabetical }}
	    <li class="site-tag">
			{{$key}},{{$value.Name}},{{$data.Plural}}
			{{$value.Page.RelPermalink}}
		<a href="/{{ $data.Plural }}/{{ $value.Name | urlize }}">
		<h3>{{ replace $value.Name "-" " " }} ({{ $value.Count }})</h3></a>
		</li>
	    {{ end }}
</ul>

👍最後註記,利用後段程式碼判斷uglyURls 的真假:

{{ $item := first 1 $.Site.RegularPages }}
{{ $link := index $item 0 }}
{{ return gt (len (findRE "\\.html$" $link.Permalink)) 0 }}

輸出所有類別

在短碼中

{{ range $taxonomyname, $x := .Site.Taxonomies }}
   {{ $taxonomyname | urlize }}
{{ end}}

結果: categories series tags

taxonomies 的範例

官網給的例子" 
<ul>
    {{ range $taxonomyname, $taxonomy := .Site.Taxonomies }}
      <li><a href="{{ "/" | relLangURL}}{{ $taxonomyname | urlize }}">{{ $taxonomyname }}</a>
        <ul>
          {{ range $key, $value := $taxonomy }}
          <li> {{ $key }} </li>
                <ul>
                {{ range $value.Pages }}
                    <li><a href="{{ .Permalink}}"> {{ .LinkTitle }} </a> </li>
                {{ end }}
                </ul>
          {{ end }}
        </ul>
      </li>
    {{ end }}
  </ul>
  • 聯結有(來自第3行):
    • <a href="localhost/categories">category</a>
    • <a href="localhost/series">category</a>
    • <a href="localhost/tags">category</a>
    • 🈸仍然不對uglyurls做反應。🐱‍🚀
  • range $value.Pages
    • 每一個類別,都視為一個section,而每個類別的值,又視為一個section所以可以拿到類別對應的.Pages(e.g. $value.Pages)

    ...🏷 md中,面板中的設定是這樣 - 『series: ["hugo"] 』

💯結論

以xx.md 的內容為例:

---
categories: ["base"]
series: ["hugo"]
tags: ["hello","lab"]
---

uglyURLs=true

  • 分別產生 http://site/categories.html,http://site/series.html,http://site/tags.html
  • base 成為key,裡面有一連串對應的網頁,xx.md是其中一個
    • 以http://site/tags.html為例
      http://site/tags.html
      • hugo 對應網頁 "./tags/hugo"
        • 🏷 和上面的差別 `.` vs `..`
      • lab 對應網頁 "./tags/lab">

uglyURLs=false

  • 分別產生 http://site/categories,http://site/series,http://site/tags,
    • 以http://site/tags(/index.html)為例
      http://site/tags 的內容
      • hugo , 對應網頁 "../tags/hugo"
        • 🏷 和上面的差別 `.` vs `..`
      • lab , 對應網頁 "../tags/lab">

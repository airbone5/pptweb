themes

layout

布景指導Hugo 怎樣產生網站。拿來主義的,可以直接在網路CLONE到themes子目錄上。

複製來的布景,如果不喜歡,可以在content同層的地方在指定一個layout子目錄,然後放上自己的設定。這樣會覆蓋`themes\下的同名檔案。

│  index.html (給home page 用的)
│
├─partials
│      custom-comments.html
│      custom-footer.html
│      custom-header.html
│      favicon.html
│      footer.html
│      header.html
│      logo.html
│      menu-footer.html
│      menu.html
│      meta.html
│      search.html
│      toc.html
│
├─shortcodes
│      directory_index_recurse.html
│      fileindex.html
│      imgur.html
│
└─_default
        list.html
        single.html  (每個section)

permalink

有關HUGO參數的存取,可以參考這裡

HUGO 的默認 permalink 方式為:

例如 content/post/linux/commandlinefu/how-to-start-tmux-as-systemd-user-service/index.md 將會生成/post/linux/commandlinefu/how-to-start-tmux-as-systemd-user-service/ 的 URL path.

如果是config.toml的設定

[Permalinks]
 /post/:filename/

,則 /post/linux/commandlinefu/how-to-start-tmux-as-systemd-user-service/ 變成了 /post/how-to-start-tmux-as-systemd-user-service/ 。

用來列出post

結論

  • 每個md都會使用_default 中的兩個檔案,section page(_index.md) 都會套用list.html,其他md(首頁例外)套用single.html
  • 首頁套用的是 \labout\index.html

where .Data.Pages "Type" "post"。

其中的post指的是contents/post 子目錄。

<!-- 
my additive 
post 是content下的一個子目錄
拿到的包括子目錄下的page

-->
    <div class="posts-list">
          {{ $pag := .Paginate (where .Data.Pages "Type" "post") }} 

          {{ range $pag.Pages }}		  
            <article class="post-preview">
			  <p> {{ print .Permalink }}</p>
              <a href="{{ .Permalink }}">
                <h2 class="post-title">{{ .Title }}</h2>
                {{ if .Params.subtitle }}
                  <h3 class="post-subtitle">
                  {{ .Params.subtitle }}
                  </h3>
                {{ end }}
              </a>
 
            </article>
           {{ end }} 
        </div>

_index.md vs index.md

index.md 是一個正常網頁,這個網頁可以有其他網頁,或是影像等等資源,所以基本上所有在同一個目錄中的檔案都是一個boundle。

_index.md 則是一個所謂的content page用來列出其他網頁。像是 home page, section page, taxonomy lists and taxonomy terms這些有子網頁的列表. 也可以包含像是images, JSON files 等等資源,但是沒有其他。而且這個boundle 限制在同一個目錄階層。

舉例,在content中有一個檔案叫做mymd.md,這個檔案在hugo的處理(uglyURLS=false)下會產生一個子目錄(mymd),然後裡面的內容會被搬到index.html中 例如

content/pages/mymd.md
若 uglyURLs=false 會變成
content
  └─pages
     └─mymd
        └─index.html

然而,當uglyURLs=false

content/pages/mymd.md
若 uglyURLs=true 會變成
content
  └─pages
     └─mymd.html

因此,在uglyURLs=false,如果要達成類似uglyURLs=true的檔案結構,作法如下:

-在某個子目錄下,例如mymd下,建立一個檔案,叫做index.md這個檔案也會變成index.html。

resource of boundle pages

  • 新建立一個子目錄(如content/pages/figtest)
    • ❓ 好像放在content 下無效?

  • 在目錄中建立一個檔案index.md,最簡單內容是,前窗(front panel)的內容:

    ---
title: 測試圖形
resources:
- { name: image1, src: images/test_bpntrain_1.png, title: Description of the image }
---

上面的目錄結構(這樣就叫做page boundle)

(content)
└─pages
   └─figtest
       │  index.md
       │
       └─images
               pic1.jpg
hugo 的翻譯(uglyURLs=true)
``` 輸出根目錄(uglyURLs=true) │ ├─pages │ │ figtest.html 📌(也就是說會根據resources中的指定,把相關影像搬到如下的目錄) │ │ │ ├─figtest📌 │ │ └─images │ │ pic1.jpg │ │ pic1_hua70eca40b85d07a93091890349b1f291_28354_1300x0_resize_q75_box.jpg │ │ pic1_hua70eca40b85d07a93091890349b1f291_28354_320x0_resize_q75_box.jpg │ │ pic1_hua70eca40b85d07a93091890349b1f291_28354_32x0_resize_q75_box.jpg │ │ pic1_hua70eca40b85d07a93091890349b1f291_28354_800x0_resize_q75_box.jpg │ │ │ └─xxx │ pic3.jpg │ ```
hugo 的翻譯(uglyURLs=false
輸出根目錄(uglyURLs=false) ├─pages │ │ │ ├─figtest📌 │ │ │ index.html📌 │ │ │ │ │ └─images │ │ pic1.jpg │ │ pic1_hua70eca40b85d07a93091890349b1f291_28354_1300x0_resize_q75_box.jpg │ │ pic1_hua70eca40b85d07a93091890349b1f291_28354_320x0_resize_q75_box.jpg │ │ pic1_hua70eca40b85d07a93091890349b1f291_28354_32x0_resize_q75_box.jpg │ │ pic1_hua70eca40b85d07a93091890349b1f291_28354_800x0_resize_q75_box.jpg │ │ │

📝 不管uglyRULs=true或false,都會有md檔案名稱對應的子目錄(eg.figtest)

🈸從上面的目錄結構,是不是可以推論utlyURLs=truefalse的時候,只要子目錄裡面有index.md,而且前版有指定resources:都可以拿到resource,❓但是若是false可以在,任何檔案( 答案是❌)

  • 把本文的內容,複製到pages子目錄的另一個檔案,然後images搬到和這個複製文件的同一個子目錄,

    └─pages
├─mytest.md
│
└─images
      20210627001800.jpg

實驗結果是,這個images子目錄的內容,找不到。💣。因為images這個子目錄沒有被搬到對應的子目錄mytest中(參看下面的目錄)。

output root(uglyURLs=false)
  └─pages
     ├──mytest(原來的mytest.md)
     │   └─index.html (mytest.md 的內容在這裡)
     ├──images

因為HUGO的預期如下:

   └─mytest
       │  index.md
       │
       └─images
               pic1.jpg
使用上限制太大,只能在一個資料夾上編輯index.md 而且同一個資料夾上的md也不會顯示。 🔑 不同MD檔案無法共用這缺點。

短碼urlimg

這裡是一個範例,用來展示怎樣存取上面的影像image

  • shortcode urlimg.html (我覺得page boundle不好用,但是,這裡的影像處理需要resource)

    {{/*
boundle page 用的
*/}}

{{ $image := .Page.Resources.GetMatch (.Get "name") }}
{{ $image32 := $image.Resize "32x" }}
{{ $image320 := $image.Resize "320x" }}
{{ $image800 := $image.Resize "800x" }}
{{ $image1300 := $image.Resize "1300x" }}

<figure>
<img class="lazy" sizes="100vw" 
src="data:image/jpeg;base64,{{ $image800.Content | base64Encode }}" 
data-srcset="{{ $image320.RelPermalink }} 320w, {{ $image800.RelPermalink }} 800w, {{ $image1300.RelPermalink }} 1300w"
{{ with $image.Title }} alt="{{.}}" {{ end }}
>
{{ with $image.Title }}
<figcaption>{{.}}</figcaption>
{{ end }}
</figure>

headless resource

某個子目錄裡面的index.md(🔑不是_index.md)，則此時這個子目錄的任何文章,都不會出現在sidebar。

----
title:
headless: true
---
說如果headless這個資料夾不會顯示在sitebar,必須是檔案index.md。如果是_index.md,則仍然顯示其中的md檔案。 以本文為例,就是在前版上加上headless=true 也許我可以記成: headless 的 page boundle。口語化一點『無頭的頁面綁定』

❓ 奇怪了咧,我發現如果把它變成一般的_index.md下面的短碼仍然可以用?

{{ $pattern := default "\\.(gif|jpg|jpeg|tiff|png|bmp)" (.Get "pattern") }}
{{ with .Site.GetPage "section" "report" }} <!-- 只拿content/report/子目錄下的資源黨-->
  {{ $resources := .Resources.ByType "image"}}
  {{ range $resources }}
    {{ with . }}
      <!--<img style="max-width: 100%;" src="{{ .RelPermalink }}">-->
        {{- $isimg := lower .Name | findRE $pattern }}
        {{- if $isimg  }}
		  {{$ext := path.Ext .Name}}
		  <img id={{.Name}} src="data:image/{{$ext}};base64,{{ .Content | base64Encode }}" > <br />
        {{- end }}
    {{ end }}
  {{ end }}
{{ end }}
